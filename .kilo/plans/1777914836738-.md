# Settings Screen Fix & Cleanup

## Issues to Fix

1. **Settings overlap**: Elements overflow the panel box and overlap each other due to insufficient vertical spacing and too many sections
2. **Remove unused settings**: FPS counter and particle quality are not needed in this game
3. **Remove audio volume**: No music or SFX exist in the game yet, so volume sliders should be removed
4. **Make working settings functional**: Fullscreen toggle and resolution change should actually apply in pygame
5. **Add "Reset Settings" button**: Restore defaults easily
6. **Keep all Estonian comments**

## Layout Analysis (Current Problem)

Panel: y=30, height=660, bottom=690
- AUDIO section (2 sliders): y=100 to y=260
- DISPLAY section (2 buttons): y=260 to y=415
- CONTROLS section (4 labels): y=415 to y=597
- GAMEPLAY section (2 buttons): y=597 to y=720 → **OVERFLOWS past panel bottom at 690**

## Fix Plan

### Clean Settings Layout (No Overflow)

Remove AUDIO section entirely (no audio in game) and GAMEPLAY section (FPS/quality not needed). Keep:
- **Display section**: Fullscreen toggle, Resolution selector
- **Controls section**: Key bindings display
- **Reset Settings button**: Restore all defaults

New spacing plan:
- panel_y = 30, panel_h = 600, panel_w = 500
- Content starts at y = panel_y + 80 = 110
- Section title padding: 35px
- Button spacing: 50px each (38px height + 12px gap)
- Controls: 28px per binding line, 20px gap between bindings

Estimated total height:
- Title area: 80px (y=30 to y=110)
- Display section: 35 + 50 + 50 = 135px (y=110 to y=245)
- Controls section: 35 + 4×28 = 147px (y=245 to y=392)
- Reset button: 50px (y=392 to y=442)
- Total content: ~412px, fits easily in panel_h=600 (bottom=630)

### Make Fullscreen Actually Work


### Make Resolution Actually Work

In `_cycle_resolution()`:

## Implementation Approach

### Screen Reference for Display Changes

The settings screen needs access to the pygame screen object to toggle fullscreen and change resolution. The cleanest approach:
1. Add `screen` parameter to `SettingsScreen.__init__()`
2. Pass `screen` when creating the screen in `game.py`
3. Use `self.screen` in `_toggle_fullscreen()` and `_cycle_resolution()`

### Save Button Behavior

- **Save button**: Actually applies the settings (fullscreen toggle, resolution change) AND persists to JSON
- **Reset Settings button**: Resets all settings to defaults AND saves
