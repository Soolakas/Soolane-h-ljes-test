# Plan: Permanent Upgrade System (Replaces Temporary Powerups)

## Overview

Replace the temporary powerup system (multi_shot_timer, speed_power_timer, rapid_fire_timer, powerup pickups from enemy drops) with a permanent upgrade system. When the player gets hit, the game pauses and presents 3 upgrade choices. Each upgrade has a rarity (Common, Uncommon, Rare, Epic, Legendary) with configurable base chances that increase over time spent without getting hit. After choosing an upgrade, chances reset and start increasing again. Upgrades persist until the player dies and restarts the run.

### New Module: `chance_scaler.py`

Handles the formula for rarity chance scaling over time. Designed to be easily tweakable for balancing.

**Formula Design:**
The formula should increase rarity chances over time without getting hit. A configurable exponential decay/sigmoid approach:

```python
# Tõenäosuste skaleerimise seaded - Chance scaling configuration
CHANCE_SCALE_RATE = 0.02  # How fast chances change per second (easily tweakable)
                          # Kui kiiresti tõenäosused muutuvad sekundis

def get_scaled_chances(elapsed_time_without_hit):
    """Calculate current rarity chances based on time since last hit.
    
    Uses a sigmoid-based scaling that increases rarity chances infinitely
    but at a decelerating rate for natural balancing.
    
    Formula: scaled_weight = base_weight * exp(-rate * time) for common
             scaled_weight = base_weight * (1 + rate * time) for higher rarities
    
    This means common decreases while others increase proportionally.
    """
    t = elapsed_time_without_hit
    rate = CHANCE_SCALE_RATE
    
    # Common decreases exponentially
    common_factor = math.exp(-rate * t)
    
    # Higher rarities increase with diminishing returns (logarithmic growth)
    uncommon_factor = 1 + rate * t * 0.5
    rare_factor = 1 + rate * t * 0.8
    epic_factor = 1 + rate * t * 1.2
    legendary_factor = 1 + rate * t * 1.5
    
    return {
        Rarity.COMMON: BASE_RARITY_CHANCES[Rarity.COMMON] * common_factor,
        Rarity.UNCOMMON: BASE_RARITY_CHANCES[Rarity.UNCOMMON] * uncommon_factor,
        Rarity.RARE: BASE_RARITY_CHANCES[Rarity.RARE] * rare_factor,
        Rarity.EPIC: BASE_RARITY_CHANCES[Rarity.EPIC] * epic_factor,
        Rarity.LEGENDARY: BASE_RARITY_CHANCES[Rarity.LEGENDARY] * legendary_factor,
    }
```

The exact formula will be refined during testing. All parameters (`CHANCE_SCALE_RATE`, multiplier factors) will be at the top of the file for easy tweaking.

## Design Notes

- The existing "UPGRADES" button in main menu is separate and unrelated (for future meta-progression/permanent unlocks across runs)
- The new upgrade system is per-run only (roguelike style)
- All formula parameters and chance configurations are at the top of their respective files for easy tweaking
- The upgrade registry is designed so new upgrades can be added by simply appending to a list
- Rarity chances scale infinitely but with diminishing returns to maintain game balance
- The system is designed to be testable and balanceable through parameter adjustment alone
