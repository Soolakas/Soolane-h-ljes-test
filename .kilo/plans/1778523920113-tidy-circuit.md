# Plan: Add Sound Effects and Volume Slider

## Overview

This game is a Pygame-based space shooter. The sound files already exist in `assets/sfx/` with descriptive names:
- `player die.wav` - Player death
- `get hit.wav` - Player taking damage
- `press start.wav` - Game start / menu action
- `enemy dead.wav` - Enemy killed
- `enemy hit.wav` - Enemy taking damage
- `Wall hit.wav` - Projectile hitting wall
- `Bullet shoot.wav` - Player shooting

The `GameSettings` class already has `sfx_volume` property (unused). The `UISlider` class already exists. The settings screen needs a volume slider added.

## Technical Details

- Use `pygame.mixer.Sound` for short effects (not `pygame.mixer.music`)
- Volume is a float 0.0 (silent) to 1.0 (full)
- Sound objects loaded once at init, cached
- Each `play()` call uses `Sound.play()` which auto-manages channels
- Volume applied via `sound.set_volume(sfx_volume)` on each play, or pre-set on load

## Issue 2: Enable concurrent sound playback

The current `SoundManager.play()` calls `self._sounds[name].play()`. By default Pygame's `Sound.play()` should allow concurrent sounds on separate channels, but the default mixer only has 8 channels which may limit simultaneous sounds.

Two fixes needed:

**a) Increase mixer channel count** in `game.py`:
```python
pygame.mixer.init()
pygame.mixer.set_num_channels(32)  # Allow up to 32 simultaneous sounds
sound_manager = SoundManager(sfx_volume=settings.sfx_volume)
```

**b) Ensure `Sound.play()` uses a fresh channel** — The default `Sound.play()` should already find an available channel, but to be explicit and ensure overlap works, the `play()` method in `sound_manager.py` should not modify the base Sound's volume directly. Instead, it should find a free channel and set the volume on that channel:

```python
def play(self, name, position=None):
    if name not in self._sounds:
        return
    
    sound = self._sounds[name]
    effective_volume = self._volume
    
    if position is not None:
        # Apply distance-based attenuation
        effective_volume = self._calculate_distance_volume(position)
    
    # Play on a new channel with the effective volume
    channel = sound.play()
    if channel:
        channel.set_volume(effective_volume)
```


Call this in `game.py` during the game loop update, before sound playback:
```python

```

**b) Add distance attenuation calculation** in `SoundManager`:
```python
def _calculate_distance_volume(self, sound_pos):
    """Calculate volume based on distance from player.
    Linear attenuation: full volume at player position, silent at max_distance."""
    if self._player_pos is None:
        return self._volume
    
    distance = self._player_pos.distance_to(sound_pos)
    max_distance = 1000  # Maximum audible distance in pixels
    
    # Linear falloff: volume decreases proportionally to distance
    distance_factor = max(0.0, 1.0 - (distance / max_distance))
    return self._volume * distance_factor
```
